SQL: the Dangers, Pros, and Cons

5/17/2014

Structured Query Language (SQL) is a programing language that is specifically used for managing data within a database system. SQL is the code used to search through your data, as well as, create, update and change data. The data in the database can be accessed using code known as queries or statements. It is a great tool for accessing and managing information, but it does come with its vulnerabilities. One issue that SQL faces is SQL Injection.

DANGERS: SQL Injection is a technique used as an attack on data bases that allows the attacker to dump the information onto their own computers, or affect the database in any other malicious way. They are able to be successful with such attacks by directly accessing and database through the web application that allows users to get information from database. While using the web application features such as login pages, search pages, and shopping carts are all legitimate ways of accessing the information needed to run a business these are also the feature that allow a SQL database to be susceptible to attack. Even firewalls and other traditional defenses do little to prevent such attacks.

SOLUTIONS: One defense against SQL injection is to use prepared statements, also known as parameterized queries. This requires the creating a template to define the SQL statements, or commands, that access the data base before they are passed into the parameters, the data, which separates data from the SQL statements or command. This lets the database recognize statements and data separately. Without this protection you could use SQL statements in a user input fields to effectively change the data base. By separating the two it makes it so a hacker cannot input statements into a data field to access the database.

SQL Databases vs NoSQL Databases:

SQL databases, normally called relational databases, use a predefined schemas and tables to represent their data in rows. NoSQL databases, normally called non-relational or distributed databases, have dynamic schema, use a key-value pairs/documents to store data and store data in terms of columns. For complex queries, or complex statements, SQL are better fit to meet those needs as it is a more complex query environment, while NoSQL often do not have as powerful of a query language to handle such requests. NoSQL databases are better for handling hierarchical data storage by using its key-value data storage. If you are looking to access and pass information to and from a database in a large volume, SQL databases are best fit, as they are stable when it comes to a large amount of transactions. Since SQL has been around for longer, it typically provides built in support from any issues that may arise within your database, while NoSQL may not have direct support and you may need to rely on the community.

SQL Query Optimization:

When working with huge databases, sifting through the information can be time consuming. In order to maximize your time it is helpful to optimize your queries. This is often done automated within a SQL program where they assess the best way of joining tables, how to handle nested SQL queries, or calculate cost estimations. Even without these tools you can use tricks to optimize your queries:

  1. When using SELECT, instead of using an asterisk to select all columns, write out each column name separated by commas.
  2. Properly using clauses. For example only us the HAVING clause as a filter and for no other function.
  3. Try to minimize redundancies in an subqueries that are being used.
  4. Follow the SQL general standard rules: use single case for all SQL verbs and begin each one on a new line.

SQL databases are prevalent and awesome. Wield their power appropriately else you may run into issues.

Object Oriented Programing Vs Functional programing

5/11/2014

Some argue that Object Oriented Programing and Functional Programing are opposites but like any tools they each have their purposes and better uses.

Here is what you need you need to know about OOP (Object Oriented Programing): It is a type of programing that have close relations between the data used and the operations that affect said data. In OOP the main actions include composes new objects and changes existing objects, or data, by way of methods. OOP, much like its title suggests, is a type of programing that is oriented towards the object it uses and creates by way of methods. A type of data defines a class and establishes how the operations will affect that data.

Here is what you need to know about Functional Programing: It is a style of programing where the data does not hold as close of a relationship with its functions. There is no straight forward definition of Functional Programming but it does have some characteristics. It is typically utilizes high order functions (HOF: takes functions as an input and outputs a function) and having minimized effects. It often uses smaller functions for a specific operation.

With this said, OOP makes it easy to change the way things are put together based on the objects of a program, which makes it ideal for programs that will be changing and evolving frequently, and where the relationship between object is important. Typically big databases use functional programing as it is efficient for treating all data similarly, while the OOP may be used to create access the data in different ways. Essentially they are not always mutually exclusive and can be combined to capitalize on their efficiencies.

JavaScript: the cool kid at web development high, but why?

4/19/2014

We are going to explore why JavaScript is so popular in web development. This is a topic that books could be written about, but I will do my best sum it up. To understand how anything successful came to be, it is important to understand where they came from.

A Bit of History

Back in 1995 Brendan Eich had the basic language design, formerly Mocha, for what today is JavaScript. Eich worked at Netscape and JavaScript was the language for Netscape Navigator. This was when Netscape Navigator was winning the browser wars, before Microsoft eventually won out with Internet Explorer. (Yes, Browser wars, and no this is not an internet reality show on TLC) Its connection to Netscape Navigator helped to establish itself. JavaScript is a language that allows for dynamic development. So, JavaScript was already hanging out with the cool kids from the start, but JS managed to stay popular, even after Netscape Navigator took the plunge into irrelevancy, because of all of its other great attributes and friends.

Its Personality

Since its infancy JavaScript has become standardized across browsers, which means that you can use it to make a web page and your web page will likely work on most, to all, browsers. Basically, It gets along with everyone. Additionally, while being powerful and fast it has become relatively easy to learn. Like anything, it take times and practice, but the amount of information about JS and it syntax on the internet makes it possible to learn with an internet connection, time and dedication. JavaScript is also easy to edit with the help of free text editors, like notepad++ of Sublime text. While this is not unique to JS it does help to add to its popularity.

JavaScripts allies

There are numerous tools that help to make JavaScript easier to use. One great tool is the Web Developer tools that opens in your browser and pulls back the curtain so you can see what is happening back stage. This makes it easier to change code, point out what needs to be fixed, or can be improved (also known as debugging). Libraries, pre- written sets of code, have also helped to keep JavaScript prominent. These libraries, such as jQuery or Pototype, help to solve complex problems by just writing a few lines of code, savings time and making it more JavaScript more friendly overall. Asynchronous JavaScript and XML (AJAX) came along and allowed for web pages to make small updates without loading the page all over again. AJAX made everything more user friends and user friendly = popular (just ask Apple).

Too many words up there, give me an over simplified break down

JavaScript is popular for web development because it got a head start by hanging out with the former popular kid; Netscape Navigator. It has great qualities such as being standard across browsers, relatively easy to learn, and easy to edit/adapt. It has gained some great allies such as Web developer Tools, libraries (jQuery), and AJAX. There are many that argue that JavaScript sucks but its hard to argue that it is not popular.

Margins, Border and Padding

April 9th, 2014

When buliding your website using HTML and CSS it is important that you don't over crowd your screen and allow for an even spacing between text, the box and the rest of the screen. For this reason it is important to understand Margins, Border and Padding. All three have a top, bottom, left and right.

Margins: this is the section that seperated your "box", also known as a section or div, from the surrounding edge of the browser window.

Border: this is what defines the edges of your "box" and gives your section (or div) a shape. The size is dictated by the "width" and "height" of the box, but the border is what can be styled and seen.

Padding: this is the space inbetween your text and the border.
